## Architecture

This package extends [Verify](https://github.com/VerifyTests/Verify) to make Serilog log output deterministic for snapshot testing.

- `VerifySerilog.Initialize()` (called from a `[ModuleInitializer]` in the consuming test assembly) is the single entry point. It registers Verify JSON converters and replaces `Log.Logger` with a `LoggerConfiguration` that has `MinimumLevel.Verbose`, `Enrich.FromLogContext`, and writes to `VerifySink`. It is single-shot — calling twice throws, and it calls `InnerVerifier.ThrowIfVerifyHasBeenRun()` to guarantee it runs before any verification.
- `VerifySink` does not buffer per-sink; it forwards every `Emit` to Verify's ambient `Recording.Add("log", logEvent)`. So output only appears in a verified file when the test calls `Recording.Start()` (or uses the equivalent scope) — sinks fire regardless, but un-recorded events are discarded.
- `Converters/` shapes Serilog types into compact JSON:
  - `LogEventConverter` emits `{ <Level>: <MessageTemplate.Text>, Properties: {...} }` and routes `Properties` through `writer.WriteMember` so Verify's scrubbers/ignores apply.
  - `LogEventPropertyConverter` flattens `{ Name: Value }`.
  - `ScalarValueConverter` unwraps the inner `.Value` (writes nothing if null, which is how empty-property output stays clean).
  - `DictionaryValueConverter` projects the Serilog `DictionaryValue` into a real `Dictionary<object, LogEventPropertyValue>` via `_.Key.Value`.
  - `PropertyEnricherConverter` uses `[UnsafeAccessor]` to read the private `_name`/`_value` fields of `Serilog.Core.Enrichers.PropertyEnricher` (no public accessors exist).
- `RecordingLogger` is a tombstone — kept only as `[Obsolete(..., true)]` redirecting callers to `VerifyTests.Recording`. Do not resurrect it.

## Conventions specific to this repo

- The Verify.Serilog assembly's namespace is `VerifyTests` (matches Verify's convention so `VerifySerilog.Initialize()` is discoverable without an extra `using`). The tests project uses `<RootNamespace>testing</RootNamespace>`.
- Global usings live in `GlobalUsings.cs` per project — prefer adding to those over per-file `using` directives.
- Strong-named (`src/key.snk`). SBOM target only runs under `CI=true`.
